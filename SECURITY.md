# Security policy

## Supported versions

Security fixes target the latest published Groove Serpent prerelease. Older development snapshots and private handoff archives are not supported releases.

## Report a vulnerability privately

After this repository has been created and private vulnerability reporting is enabled, use the
repository's private advisory form:

<https://github.com/BluntforceRiot/groove-serpent/security/advisories/new>

If that form is unavailable, the project does not currently publish an alternate private reporting
channel. Retain the report until this policy names a working private channel; do not guess an email
address or send vulnerability details through a public channel.

Do not open a public issue, discussion, or pull request for a suspected vulnerability, and do not post exploit details publicly. Include a concise impact description, affected version or commit, a minimal synthetic reproduction, and any suggested mitigation in the private report.

Do not include copyrighted recordings or artwork, API keys, credentials, private project files, collection metadata, or local paths. Use synthetic fixtures and redact personal information.

The maintainers will coordinate status and any next steps through the private advisory.
