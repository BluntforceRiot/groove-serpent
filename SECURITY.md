# Security policy

## Supported versions

Security fixes target the latest `1.0.x` Groove Serpent release. The historical
`0.5.0-alpha.1` prerelease, development snapshots, and private handoff archives are not supported
releases.

## Report a vulnerability privately

This repository has private vulnerability reporting enabled. Use its private advisory form:

<https://github.com/BluntforceRiot/groove-serpent/security/advisories/new>

If that form is temporarily unavailable, the project does not publish an alternate private
reporting channel. Retain the report until the advisory form is available; do not guess an email
address or send vulnerability details through a public channel.

Do not open a public issue, discussion, or pull request for a suspected vulnerability, and do not
post exploit details publicly. Include a concise impact description, affected version or commit, a
minimal synthetic reproduction, and any suggested mitigation in the private report.

Do not include copyrighted recordings or artwork, API keys, credentials, private project files,
collection metadata, or local paths. Use synthetic fixtures and redact personal information.

The maintainers will coordinate status and any next steps through the private advisory.
